concepts

namespaces

Text and files live in separate namespaces so their keys never collide:

URLtype
/key/clipboard (text)
/f/key/file

/notes/ is a clipboard. /f/notes/ is a file. They're completely independent — same key name, different namespaces.

reserved keys

Some keys are reserved by the system and cannot be used for drops — for example help, auth, admin, billing. These correspond to top-level URL paths used by the application. The key availability check will show them as taken, and the server will reject them at save time.

locking

Anonymous drops get a 24-hour creation lock. Nobody can overwrite or rename the key during that window. After 24h, anyone with the key can replace the content.

Paid drops are permanently locked to your account. Only you can edit, rename, delete, or renew them. Nobody else can overwrite a key you own.

saved drops

You can bookmark any drop you didn't create — useful for drops shared with you. Saved drops appear in your account dashboard and are included in exports. Saving grants no edit or delete rights.

anonymous drop claiming

Drops you create before signing up are tracked by a browser cookie. When you register or log in, those drops are automatically claimed and transferred to your account, with their lifetime extended to free-tier limits.
